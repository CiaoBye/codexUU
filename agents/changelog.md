本文件记录本地复刻项目的每次有效更新。每次代码更新完成后，需同步维护根目录 `AGENTS.md` 与本文件，并重启程序。

## 2026-07-14 · 0.1.08 · 原生窗口修复、额度通道收敛与圆形状态窗

### 必须修复

- 移除羊毛进度的低频涌动高光循环，仅保留一次性的数值/进度缓动，避免暗色界面中的不连续亮线与空闲定时器开销。
- 轻量模式不再把主窗口改为 `Qt.Tool`：主窗口恢复 Windows 标准最小化、最大化、关闭按钮；仅通过 `WS_EX_TOOLWINDOW` 尝试隐藏任务栏入口，并在深色/浅色主题切换后重新应用 DWM 标题栏颜色。仪表盘顶部重复的关闭按钮已移除。
- 设置存在未保存草稿时，关闭或取消将先提示确认；桌面悬浮样式也纳入“保存设置后生效”的草稿流程。

### 数据与桌面体验

- 核验本机 Windows Store 版 Codex：其 `WindowsApps` 内嵌 `codex.exe` 被子进程启动策略拒绝，不能可靠地作为 stdio app-server。额度因此使用最新 session 中官方写入的 `rate_limit` 快照；读取仅扫描最近 14 天、按修改时间最多 32 个 rollout 文件，并复用文件级解析缓存，避免全历史扫描和刷新阻塞。独立 CLI 可用时仍优先 app-server。
- 数据源诊断明确区分实时 app-server 与 Store 桌面版 session 快照回退，不再把受限安装误报成“执行别名”。
- 桌面状态窗重构为圆形信息窗：提供“圆环 / 光晕双环 / 极简圆形”三种持久化样式，显示当前 Runtime 的真实可用额度、今日 Token 与重置时间；移除横向卡片、打开/隐藏按钮，改为拖动定位、双击打开主窗口与 hover 详情。
- 手动 GitHub Actions 构建无论是否发布 Release 都上传安装器及 SHA-256 Artifact（保留 14 天）；日常推送不触发构建或发布。

- `PYTHONPATH=. python -m pytest -q`：44 项通过。
- Qt offscreen：验证主窗口仍为普通 `Qt.Window`、三种圆形悬浮样式尺寸可创建。
